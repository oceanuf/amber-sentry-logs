### 🎯 整改目标
解决市场成交卡片涨跌个股数使用固定估计值的问题，实现基于市场实际情况的动态智能估算。

### 🛠️ 技术方案
**方案选择**: 智能估算算法（因Tushare Pro标准权限无直接up_count/down_count字段）

**算法原理**:
1. **数据基础**: 获取真实A股数量（上海1703 + 深圳2300 = 4003只）
2. **市场情绪**: 基于上证指数涨跌幅分类
   - 大跌日(>2%): 下跌个股80-90%
   - 中跌日(1-2%): 下跌个股70-80%
   - 小跌日(0.5-1%): 下跌个股60-70%
   - 微跌日(0-0.5%): 下跌个股55-60%
   - 微涨日(0-0.5%): 下跌个股45-55%
   - 小涨日(0.5-1%): 下跌个股35-45%
   - 中涨日(1-2%): 下跌个股25-35%
   - 大涨日(>2%): 下跌个股15-25%

3. **动态计算**:
   ```
   下跌比例 = base_ratio + adjustment_factor * |涨跌幅|
   上涨个股数 = 总A股数 × (1 - 下跌比例)
   下跌个股数 = 总A股数 × 下跌比例
   ```
