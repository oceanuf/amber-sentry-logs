## 📋 项目概述

### 问题总结
基于2026-03-19琥珀引擎v3.2.6截图分析，发现以下核心问题：

1. **数据不一致性** (严重)
   - 沪深300：智能卡片4658.33 vs Tushare实际4583.25 (-1.61%)
   - 三重数据源并存，无统一权威
   - 智能卡片与右侧普通卡片数据不同步

2. **颜色习惯混淆** (高优先级)
   - CSS定义：`.price-up`绿色(#4caf50) / `.price-down`红色(#f44336) - **国际习惯**
   - HTML内联：`.change-up`红色(#E74C3C) / `.change-down`绿色(#2ECC71) - **中国习惯**
   - 创业板指显示-1.11%却使用`price-up`类名

3. **更新机制碎片化** (中等)
   - 单点更新仅处理右侧普通卡片
   - 智能卡片、宏观双锚数据静态化
   - 依赖手动脚本，无自动化流程

4. **布局风险** (中等)
   - 智能卡片无高度限制，可能再次出现V3.2.4视觉崩塌
   - 响应式设计测试不足

### 项目目标
1. **数据归一化**：建立Tushare Pro单一权威数据源
2. **视觉标准化**：全站统一应用中国红涨绿跌习惯
3. **自动化更新**：收盘后自动同步所有模块
4. **布局加固**：预防视觉崩塌，确保响应式兼容

## 🏗️ 架构设计

### 数据架构：单一权威管道
```
Tushare Pro (权威数据源)
      ↓
Amber Data Engine (数据加工与验证)
      ↓
    ├── 智能卡片模块
    ├── 普通指数卡片  
    ├── 宏观双锚仪表盘
    ├── 估值云图
    └── 数据缓存 (24小时TTL)
```

### 技术栈
- **数据源**: Tushare Pro API (2120积分，标准研究级权限)
- **处理层**: Python 3.12 + pandas + SQLite缓存
- **展示层**: HTML5 + CSS3 (V2.2琥珀色视觉系统)
- **自动化**: Cron定时任务 + 错误重试机制
- **监控**: 日志系统 + 健康检查
