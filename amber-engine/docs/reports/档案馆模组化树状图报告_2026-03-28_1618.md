## 🔧 技术实现详情

### 1. 数据挂载机制 (核心创新)
**问题**: Cheese之前直接在MD里写死净值，违反"极效管理"原则
**解决方案**: 实现"模版+数据注入"架构

#### 标的详情模版示例 (`518880_Gold.md`):
```markdown
---
ID: ASSET-001
TYPE: Gold_ETF
CODE: 518880.SH
STATUS: Holding
WEIGHT: 0.35
---

# 🥇 518880 - 黄金ETF

## 📊 净值表现
[[VALUATION_ZONE]]  <!-- PHP注入锚点 -->

## 📈 技术指标
- 30日涨幅: {{30d_return}}%
- 年化波动率: {{annual_volatility}}%
```

#### PHP数据注入逻辑:
```php
// 数据注入逻辑
$asset_data = json_decode(file_get_contents('database/518880.json'), true);
$valuation_html = generate_valuation_table($asset_data);
$markdown = str_replace('[[VALUATION_ZONE]]', $valuation_html, $markdown);
```

### 2. YAML配置标准化
所有MD文件头部必须包含结构化元数据:

```yaml
---
ID: STELLAR-01
TYPE: Trend_Following
HOLD_PERIOD: 60D
STATUS: Active
VERSION: V1.1.1-ELITE
CREATED: 2026-03-20
UPDATED: 2026-03-28
---
```

**价值**: 便于PHP母模版自动生成"档案馆索引目录"，无需手动维护index文件。

### 3. PHP母模版升级
`index.php`已更新支持四模组架构:

```php
// 导航菜单更新
```
