# 📊 数据模版库 - Database目录

## 📋 目录规范
基于[2614-001]档案馆单体仓库同步法典 V3.1

### 允许文件类型
- **仅允许 `.json` 结构定义文件**
- 严禁同步真实资金与敏感持仓数据
- 仅作为数据结构模板和占位符

### 文件命名规范
```
{模块}_{用途}_template.json
示例:
- etf_structure_template.json
- portfolio_template.json
- audit_result_template.json
```

## 🚫 严格禁止
- ❌ 真实资金数据
- ❌ 敏感持仓信息
- ❌ 个人账户信息
- ❌ 交易记录详情
- ❌ 任何形式的私密数据

## ✅ 允许内容
- 数据结构定义
- 字段类型说明
- 示例数据 (脱敏后)
- 配置模板
- 接口响应格式

## 📁 示例文件结构
```json
{
  "template_name": "ETF数据结构模板",
  "version": "1.0.0",
  "description": "仅作为数据结构参考，禁止填充真实数据",
  "fields": {
    "code": "标的代码 (示例值)",
    "name": "标的名称 (示例值)",
    "nav": 1.0,
    "date": "2026-01-01"
  },
  "security_notice": "本文件仅包含示例数据，真实数据禁止同步至GitHub"
}
```

## 🔒 安全审查
所有JSON文件必须：
1. 不包含真实数值 (仅示例值)
2. 不包含个人身份信息
3. 通过 `scripts/github/sync_clean.sh` 安全扫描
